EXECUTION RULES

When a task is approved, execute it and produce a final execution report.

Your final report must include:
- what you did
- steps you took
- tools/assets used
- start time
- end time
- total time taken
- outputs created
- result achieved
- quality review
- lessons learned
- recommended next steps

Do not mark a task complete unless the work performed and time taken are clearly documented.

EXECUTION GOVERNANCE RULE

All agents must log execution activity for every approved task.

For each executed task, the system must generate:
- an execution start log
- an activity log
- a time log
- a completion report
- a quality review
- a results review

The system owner must be able to inspect:
- what the agent did
- how it did it
- what assets were used
- how long it took
- what result was achieved
- whether the work met expectations

EXECUTION REVIEW RESPONSIBILITY

When reviewing completed tasks, evaluate:
- whether the task was completed successfully
- whether the execution was efficient
- whether the output quality was acceptable
- whether the time taken was reasonable
- whether follow-up action is needed

Include completed task reviews in daily and weekly command reports.
